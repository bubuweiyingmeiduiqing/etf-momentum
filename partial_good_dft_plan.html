SSN 架构下 Partial Good DFT 测试方案

目标：坏 Tile 不从 SSN 拿掉；不改 Scan / EDT / MISR 拓扑；一次 ATPG、一次 WGL、一次 STIL 支持所有 Partial Good SKU。

核心：所有 Tile 永远参与同一套 SSN 扫描加载/卸载和同一套 ATPG pattern；SKU 差异只由运行时 Tile Good/Bad Mask 与 On-Chip Compare Mask 决定。

1. 目标与约束

必须满足

  • SSN / Streaming Scan Network 结构。
  • 支持 On-Chip Compare。
  • 坏 Tile 不从 SSN endpoint list 中移除。
  • 一次 ATPG、一次 WGL、一次 STIL。
  • 同一套 pattern 支持所有 Partial Good SKU。

不能改变

  • 不能改 Scan chain topology。
  • 不能改 EDT decompressor / compactor topology。
  • 不能改 MISR polynomial、width、input mapping、feedback。
  • 不能为不同 SKU 生成不同 ATPG pattern。
一句话方案：用 Full Good 模型生成唯一 ATPG pattern，所有 Tile 固定留在 SSN / Scan / EDT / MISR 拓扑中；每个 Tile 后加固定 On-Chip Compare 和 fail sticky，通过 IJTAG/ATE 加载 Partial Good SKU mask，只 mask pass/fail 判定，不改任何 DFT 拓扑。

3. 关键设计原则

3.1 坏 Tile 不从 SSN 中移除

SSN endpoint / packet route / schedule 固定。即使某个 Tile 已知坏，也仍然接收 SSN packet、执行 scan shift/capture/unload，并保持与其它 Tile 相同的 SSN 调度。

SSN endpoint list:
  Tile0 endpoint: always present
  Tile1 endpoint: always present
  Tile2 endpoint: always present
  ...
  TileN endpoint: always present

3.2 不改 Scan / EDT / MISR 拓扑

模块固定内容禁止做法
Scanchain count、chain order、stitchingBad Tile bypass、per-SKU chain removal、per-SKU chain re-stitch
EDTchannel count、decompressor、compactor、mappingper-SKU EDT remap、per-SKU decompressor/compactor change
MISRpolynomial、width、feedback、input mapping、resetper-SKU MISR polynomial 或 input map 变化
SSNendpoint、packet map、route、schedule从 SSN 中删除坏 Tile endpoint
Partial Good 只改变“这个 Tile 的 response 是否参与 pass/fail 判断”，不改变 scan stimulus、SSN packet、EDT 解压/压缩和 MISR 拓扑。

4.2 Comparator 逻辑

compare_mismatch =
    pattern_valid
  & tile_compare_enable
  & compare_mask
  & (actual_response ^ expected_response);

always_ff @(posedge test_clk or negedge test_rst_n) begin
  if (!test_rst_n) begin
    tile_fail_sticky[i] <= 1'b0;
  end else if (clear_tile_fail[i]) begin
    tile_fail_sticky[i] <= 1'b0;
  end else if (compare_mismatch) begin
    tile_fail_sticky[i] <= 1'b1;
  end
end

4.3 Final Fail Aggregation

assign effective_tile_fail[i] =
       pg_tile_required_mask[i]
     & tile_compare_enable[i]
     & tile_fail_sticky[i];

assign final_fail = |effective_tile_fail;
assign final_pass = ~final_fail;

5. Partial Good SKU Mask 与 Binning

5.1 建议寄存器

建议通过 IJTAG / TAP / SSN sideband / test register 加载以下寄存器：

PG_TILE_REQUIRED_MASK[N:0]
PG_TILE_COMPARE_ENABLE[N:0]
PG_TILE_KNOWN_BAD_MASK[N:0]
PG_BIN_MODE
PG_MIN_GOOD_TILE_COUNT

5.2 SKU 示例

假设 8 个 Tile，bit 顺序为 Tile7 ... Tile0：

SKU要求Mask 示例
Full Good所有 Tile 必须 pass8'b1111_1111
SKU_A允许 Tile7 坏8'b0111_1111
SKU_B允许 Tile3、Tile6 坏8'b1011_0111
SKU_C任意 6 个 Tile 好即可使用 count-based binning：good_count >= 6

5.3 Count-Based SKU

assign tile_pass[i] = ~tile_fail_sticky[i];

good_tile_count = popcount(tile_pass);

assign final_pass_count_mode =
    good_tile_count >= pg_min_good_tile_count;

5.4 Hybrid SKU

required_fail = |(pg_tile_required_mask & tile_fail_sticky);
optional_good_count = popcount(pg_optional_tile_mask & ~tile_fail_sticky);

final_pass_hybrid =
    ~required_fail &&
    optional_good_count >= pg_optional_min_good_count;

7. 一次 ATPG / 一次 WGL / 一次 STIL 策略

7.1 ATPG 按 Full Good 建模

All Tile exist.
All scan chains exist.
All EDT channels exist.
All MISR/compactor exist.
All SSN endpoints exist.
All Tile faults are targeted.

ATPG 不为每个 SKU 单独生成 pattern。唯一 pattern set 是：

Pattern set = P_all_tiles

7.2 Same Pattern, Different Mask

Same pattern:
  scan stimulus
  scan unload
  expected response
  compare mask
  clocking
  SSN schedule

Different SKU:
  pg_tile_required_mask
  pg_bin_mode
  pg_min_good_tile_count

7.3 STIL/WGL 组织建议

partial_good_all_sku.stil
  ├── Header
  ├── Signals
  ├── Timing
  ├── ScanStructures
  ├── Procedures
  │     ├── ssn_init
  │     ├── edt_init
  │     ├── occ_init
  │     ├── misr_reset
  │     ├── pg_clear_fail_sticky
  │     ├── pg_load_mask_full_good
  │     ├── pg_load_mask_sku_a

  │     ├── pg_load_mask_sku_c

        ├── pattern_0

ATE program 只选择不同的 mask-load procedure；common scan pattern body 不变。因此仍是一份 STIL、一份 WGL、一套 ATPG pattern。

8. MISR 污染风险与处理

这是本方案最关键风险：如果坏 Tile 的未知 response 已经进入共享 MISR，则 MISR signature 被污染，后面再 mask 该 Tile 也可能来不及。

8.1 推荐方案 A：On-Chip Compare 在 MISR 判定之前完成

Tile scan response
      │
      ▼
On-Chip Compare
      │
      ▼
tile_fail_sticky
      │
      ▼
PG mask final fail

MISR 仍然可以存在，但 final pass/fail 不依赖全局共享 MISR signature，而依赖 per-Tile compare sticky。

8.2 推荐方案 B：固定 response mask gate，MISR 拓扑不变

misr_input[i] =
    tile_compare_enable[i] ? actual_response[i]
                           : expected_response[i];

或：

misr_input[i] =
    tile_compare_enable[i] ? actual_response[i]
                           : 1'b0;

前提是 ATPG / MISR reference 也按同样 mask 规则处理。

  • MISR polynomial 不变。
  • MISR width 不变。
  • MISR feedback 不变。
  • MISR input pin 数不变。
  • 只是固定地在输入前有一个 mask mux；per-SKU 只改变 mux select，不改变拓扑。

10. 验证 Checklist

10.1 拓扑一致性检查

  • Scan chain count / order 在所有 SKU 完全一致。
  • EDT channel count、decompressor、compactor、mapping 完全一致。
  • SSN endpoint list、packet route、schedule 完全一致。
  • MISR polynomial、width、input mapping、reset sequence 完全一致。
  • ATPG pattern set、WGL、STIL 唯一。

10.2 Mask 功能仿真

Case条件预期
1Tile i good，mask = 1Final pass
2Tile i bad，mask = 1Final fail
3Tile i bad，mask = 0，其它 required Tile passFinal pass
4多个 Tile bad，部分 masked，部分 unmasked只有 unmasked bad Tile 导致 fail

10.3 需要和 Siemens AE / 内部 DFT Owner 确认

  1. On-Chip Compare expected data 来源：从 ATE/SSN stream 送，还是片上生成？
  2. MISR 是否参与最终 pass/fail；如果参与，坏 Tile response 是否会污染 signature？
  3. SSN pattern export 是否支持 compare annotation、expected/mask/endpoint mapping。
  4. IJTAG PDL 是否被合入 WGL/STIL，PG mask load sequence 是否可由 ATE 选择。
  5. Partial Good binning 在片上做，还是由 ATE 根据 tile fail bitmap 做。
  6. ATPG 是否对所有 Tile fault 一次性 target，且没有 per-SKU fault removal。
最推荐落地方式：片上只产生 tile_fail_bitmap / good_count / optional final_fail；ATE 根据 bitmap 做 SKU binning。这样一套 pattern 覆盖全部 Tile，Partial Good 规则可在 ATE 侧灵活调整。
