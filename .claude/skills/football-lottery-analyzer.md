# 足球彩票AI分析助手

用 AI 辅助分析足球比赛，基于赔率、市场数据、Forebet/PredictZ 预测、历史对战、近期状态等多维度信息，给出胜平负和让球胜平负建议。

## 触发条件

当用户提供足球比赛信息（赔率、球队、联赛等）并请求分析时，使用此技能。

## 输出文件保存（必须执行）

**分析完成后，必须将完整报告保存为 PDF 文件：**

- **保存路径**：`D:\mycode\football\{日期}-{赛事}-{主队}vs{客队}-分析报告.pdf`
- **转换工具**：`D:\mycode\football\md2pdf.py`（使用 fpdf2 将 Markdown 转 PDF）
- **生成流程**：
  1. 先输出分析结果给用户查看
  2. 将完整报告内容（Markdown 格式）写入临时 `.md` 文件
  3. 调用 `python D:\mycode\football\md2pdf.py "<md文件路径>" "<pdf输出路径>"` 生成 PDF
  4. 删除临时 `.md` 文件（仅保留 PDF）
  5. 告知用户 PDF 文件路径
- **目录处理**：若 `football` 文件夹不存在，先创建再写入
- **文件命名规则**：
  - 单场：`2026-06-12-世界杯-墨西哥vs南非-分析报告.pdf`
  - 多场（同一赛事）：`2026-06-12-世界杯-两场分析报告.pdf`
  - 多场（不同赛事）：用赛事简称拼接，如 `2026-06-12-世界杯+英超-三场分析报告.pdf`
- **⚠️ 重要**：先输出分析结果给用户查看，再保存 PDF。保存完成后告知用户文件路径。

## 输入要求

用户需要提供以下信息（可分批提供）：
- **比赛信息**：联赛、主队名称、客队名称、比赛日期
- **国内足彩赔率**：胜平负赔率（主胜/平/客胜）、让球胜平负赔率（含让球数）
- **玩法制约**：用户最多串 2 场，需要明确是单场还是 2 串 1

若用户只给赔率、未给完整比赛信息，先根据可用信息分析；无法确认赛事时再追问，不要编造对阵或赔率。

## 分析流程

### 第一步：搜集数据

针对每场比赛，同时进行以下搜索，并在报告中标注关键来源与数据时间：

1. **近期状态（最近10场比赛）**
   - 搜索 `"[主队名] 最近10场比赛 战绩 2026"` 和 `"[客队名] 最近10场比赛 战绩 2026"`
   - 关注：胜平负走势、进球数、失球数、主客场表现差异
   - 提取近10场胜率、不败率、场均进球、场均失球

2. **历史对战（Head-to-Head）**
   - 搜索 `"[主队名] vs [客队名] 历史交锋 战绩"`
   - 关注：总交锋记录、主队主场交锋记录、最近5次交锋结果
   - 注意是否有“克星”关系（一方对另一方有明显心理优势）

3. **伤停/轮换信息**
   - 搜索 `"[主队名] 伤停 首发 预测 2026"` 和 `"[客队名] 伤停 首发 预测 2026"`
   - 关注核心球员缺阵、周中是否有欧战消耗、轮换幅度

4. **主流预测网站：Forebet 与 PredictZ（必须尝试）**
   - 搜索 `site:forebet.com "[主队名]" "[客队名]" prediction` 和 `site:predictz.com "[主队名]" "[客队名]" prediction`
   - 提取：预测赛果、比分、胜平负倾向、进球数倾向（如 Over/Under、BTTS）
   - 若两个网站意见一致，提高“主流预测共识”评分；若分歧明显，降低置信度并说明分歧点
   - 若网站无法访问或无该场数据，明确写“未获取到可靠数据”，不得臆造

5. **国内足彩赔率分析**
   - 根据用户提供的胜平负/让球胜平负赔率，计算隐含概率：`隐含概率 = 1 / 赔率`
   - 计算返还率与归一化概率：
     - `返还率 = 1 / (1/主胜赔率 + 1/平赔率 + 1/客胜赔率)`
     - `归一化概率 = 单项隐含概率 / 三项隐含概率之和`
   - 关注赔率变化趋势（初盘 vs 即时盘，若用户提供）

6. **Polymarket 市场对比（当用户提供国内足彩赔率时必须执行）**
   - 搜索 `Polymarket [主队名] [客队名]`、`site:polymarket.com [主队名] [客队名]`，匹配同一赛事与开赛时间
   - 提取 Polymarket 的市场价格/概率（主胜、平局、客胜；若只有晋级/冠军/双项市场，必须说明不可直接等同胜平负）
   - 将 Polymarket 概率换算为公平赔率：`公平赔率 = 1 / Polymarket概率`
   - 对比国内足彩归一化概率与 Polymarket 概率，给出：
     - 概率差：`国内归一化概率 - Polymarket概率`
     - 赔率差：`国内赔率 - Polymarket公平赔率`
     - 价值判断：国内赔率是否高于市场公平赔率，是否存在正向价差
   - 分析 Polymarket 的流动性、成交量、盘口是否临近失真；若流动性低，降低参考权重

### 第二步：多维度评分

对每场比赛按以下维度打分（1-10分制），评分必须融入 Forebet、PredictZ 与 Polymarket 对比结果：

| 维度 | 权重 | 评分依据 |
|------|------|----------|
| 主队近期状态 | 15% | 近10场胜率、进攻/防守数据 |
| 客队近期状态 | 14% | 近10场胜率、进攻/防守数据 |
| 主客场与赛程 | 13% | 主队主场战绩、客队客场战绩、体能/赛程 |
| 历史交锋 | 8% | H2H 记录，主队历史优势 |
| 伤停影响 | 12% | 核心球员缺阵情况 |
| 战意/阵容动机 | 10% | 联赛压力、杯赛战意、轮换可能 |
| 国内赔率价值 | 10% | 国内赔率是否高于合理估值，返还率与赔率结构 |
| Polymarket市场对比 | 10% | 与 Polymarket 概率/公平赔率的偏差、流动性 |
| Forebet/PredictZ共识 | 8% | 两站预测方向、比分/进球倾向是否一致 |

综合评分 = 加权平均分 × 10 → 百分制。

### 第三步：AI综合判断

结合所有数据，给出以下判断：

1. **胜平负建议**：主胜 / 平局 / 客胜 / 放弃
   - 置信度（高/中/低）
   - 核心理由最多 3 条，避免冗长

2. **让球胜平负建议**：让球主胜 / 让球平 / 让球客胜 / 放弃
   - 简要说明让球数是否合理、能否穿盘
   - 若让球盘与胜平负方向冲突，优先提示风险而不是强行推荐

3. **风险评估**：
   - 🟢 低风险 / 🟡 中风险 / 🔴 高风险
   - 关键风险因素最多 3 条

### 第四步：投注建议精简规则

投注建议要“短、明确、可执行”，只保留必要结论：

- 单场最多给 **1 个主推 + 1 个备选/防守项**；没有价值时直接建议“放弃”。
- 不再输出长篇投注理由，理由压缩为 2-3 条。
- 不给激进倍投、追损、重仓建议。
- 若主推赔率低于 1.30，默认提示性价比不足；除非市场对比显示明显低估，否则不推荐。
- 若信息不足、Polymarket 流动性低、Forebet/PredictZ 分歧大，降低置信度或建议跳过。

### 第五步：串关组合建议

如果用户提供多场比赛（≥2场），给出最佳 2 串 1 组合建议，但保持精简：

- **推荐组合**：最多 1 组 2 串 1
- **组合逻辑**：一句话说明（如：一场稳胆 + 一场价值项）
- **预计综合赔率**：两场赔率乘积
- **资金纪律**：提醒串关风险高，建议小注，不超过预算的 5%-8%

## 重要原则

1. **数据优先**：所有判断必须有数据支撑，不能凭空猜测。
2. **市场对比优先**：用户提供国内足彩赔率时，必须对比 Polymarket；无法获取时要明确说明并降低市场判断权重。
3. **融入主流预测**：评分时必须考虑 Forebet、PredictZ；两者分歧时降低置信度。
4. **投注建议精简**：建议短而清晰，最多 1 个主推 + 1 个备选；没有价值就建议放弃。
5. **承认不确定性**：对于信息不足的比赛，降低置信度，甚至建议放弃。
6. **资金纪律**：始终提醒用户控制投注金额、设置止损。
7. **反对追热**：赔率过低的选项（<1.30）不具备购买价值，提醒用户性价比不足。
8. **串关提示**：串关放大了体彩约27%的抽水劣势，提醒用户串关的实际期望值低于单场。
9. **走水提醒**：让球盘中，让球数为整数时存在走水可能，需提前说明。
