# ETF多资产全球轮动策略 — 量化专家复盘与提示词优化

## 概述

这是一个"元分析"提示词系统。它不是用来生成每日交易建议的，而是**站在更高维度，以顶级量化专家的身份，审视策略的历史表现，诊断提示词质量，给出优化方案**。

## 适用时机

建议每 **4次调仓（约1个月）** 执行一次策略复盘。

## 文件说明

| 文件 | 用途 |
|------|------|
| `strategy_review_prompt_config.json` | 完整配置：专家system prompt + 复盘模板 + 数据Schema |
| `strategy_review_prompt_template.txt` | 独立提示词模板，替换 `{REVIEW_DATA}` 后直接发送 |
| `strategy_review_data_template.json` | 复盘数据输入模板，填入调仓/止损/收益统计数据 |

## 复盘分析6大模块

| 模块 | 核心问题 |
|------|----------|
| **📊 收益事实评估** | 策略赚了多少？vs基准如何？夏普/卡玛是否合格？收益来源是什么？ |
| **🔍 信号系统诊断** | SMA20过滤有效吗？风险调整得分能预测收益吗？溢价过滤对了吗？ |
| **🛡️ 风控系统诊断** | 波动率截断降低回撤了吗？两种止损哪种更有效？风险平价优于等权吗？ |
| **📋 提示词诊断** | 逻辑一致吗？有模糊表述吗？缺失了什么？有指令冲突吗？ |
| **💡 优化建议** | 3-5条具体优化方案，含原文引用、数据支撑、修改文本、预期效果、优先级 |
| **📌 复盘总结** | 核心结论 + 行动清单 + 观察项 + 策略信心评级 |

## 专家角色设定

提示词内置了一个"顶级量化对冲基金CSO"人设：
- **数据驱动**：每个结论必须用统计数字支撑
- **冷酷诚实**：不美化业绩，不回避问题
- **建设性批判**：每个批评附带具体改进方案
- **过拟合警觉**：警惕历史表现好但逻辑脆弱的信号
- **提示词工程视角**：评估策略描述本身的精确性

## 使用流程

1. 汇总最近4次调仓的实际交易数据（收益、止损事件、过滤统计等）
2. 填入 `strategy_review_data_template.json`
3. 替换 `strategy_review_prompt_template.txt` 中的变量
4. 配合 `strategy_review_prompt_config.json` 中的 system_prompt 发送给大模型
5. 大模型输出 HTML 复盘报告，包含具体优化建议

## 与日报提示词的关系

```
日报提示词 (momentum_daily_prompt_template_v2.txt)
    ↓ 每日/每周运行，生成交易建议
    ↓
实际交易执行 → 产生收益/止损数据
    ↓
复盘提示词 (strategy_review_prompt_template.txt)
    ↓ 每月运行一次
    ↓
诊断日报提示词的问题 → 给出优化建议
    ↓
人工审核后 → 更新日报提示词
    ↓
循环迭代
```
