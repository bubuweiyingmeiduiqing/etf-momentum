# ETF动量策略量化日报 — 结构化提示词系统

## 概述

本系统提供一套结构化的量化分析提示词，用于在**自行计算**ETF均价、动量数据、波动率数据和前期资金投入情况后，将这些结构化数据输送给大模型（DeepSeek / GPT-4 / Claude 等），生成基于动量策略的量化日报。

## 文件说明

| 文件 | 用途 |
|------|------|
| `momentum_prompt_config.json` | 完整配置：含数据输入schema、提示词模板、评分指南、交易规则参考 |
| `momentum_daily_prompt_template.txt` | 独立提示词模板，直接替换 `{DATA_INPUT}` 和 `{REPORT_DATE}` 后发送给大模型 |
| `momentum_data_input_template.json` | 数据输入模板，按结构填入你计算的量化数据 |

## 使用流程

### 第一步：自行计算ETF量化指标

你需要计算以下数据（参考 `momentum_prompt_config.json` 中的 `momentum_scoring_guide` 评分体系）：

**价格数据：**
- 最新收盘价、前日收盘价、开盘价、最高价、最低价
- 成交量、成交额、涨跌幅
- 持仓均价（如有持仓）

**今日动量数据：**
- 5日/10日/20日收益率
- SMA5 / SMA20 及其方向（上行/走平/下行）、斜率百分比
- 成交量 vs 5日均量比值、vs 前日比值
- 是否为阳线、收盘价距SMA5的偏离%
- 动量评分（0-100）及等级

**前期动量数据（上一交易日）：**
- 前期的5日/10日收益率
- 前期的量能比值
- 前期动量评分及等级
- 动量趋势变化判断（转强/转弱/延续）

**波动率数据：**
- 5日/20日波动率
- 14日ATR及ATR%
- 布林带上轨/中轨/下轨及当前价格位置
- 20日最大回撤

**信号检测（4大信号）：**
- ①反抽入场：前日阴线+今日阳线 & 价在SMA5±4.5% & 量≥前日1.45倍
- ②动量修复：近5日收益转正/连正 & 近10日向上修复
- ③均线突破：收盘站上SMA20 & SMA20走平/上翘
- ④放量阳线：量>5日均量1.35倍 & 阳线

**前期资金投入：**
- 上期持仓品种、金额、仓位%、入场价、止损价
- 当前浮盈/浮亏
- 可用资金、总资金池

### 第二步：填入数据模板

将计算结果填入 `momentum_data_input_template.json`，得到完整的 `DATA_INPUT` JSON。

### 第三步：组装提示词并发送

将 `momentum_daily_prompt_template.txt` 中的：
- `{REPORT_DATE}` 替换为当前日期
- `{DATA_INPUT}` 替换为第二步得到的完整JSON

然后发送给大模型（附上 system prompt）。

### 第四步：获取HTML日报

大模型将返回标准化的 HTML 格式量化日报，包含6个模块：
1. 📊 今日行情概览
2. 📈 动量分析（核心）
3. 🌊 波动率分析
4. 🎯 信号系统检查
5. 💰 持仓建议与资金配置
6. ⚠️ 风险提示与关键观察点

## 动量评分参考体系

| 分项 | 权重 | 评分规则 |
|------|------|----------|
| 5日收益率 | 30% | >2%=30, 0~2%=20, -1~0%=10, <-1%=0 |
| 10日收益率 | 20% | >1%=20, -2~1%=10, <-2%=0 |
| SMA5方向 | 15% | 上行=15, 走平=8, 下行=0 |
| SMA20方向 | 15% | 上行=15, 走平=8, 下行=0 |
| 量价配合 | 10% | 放量阳线=10, 缩量阳线=5, 放量阴线=0 |
| 收盘vs SMA20 | 10% | 站上=10, 之下=0 |

**等级划分：** ≥80强势 / 60-79中等偏强 / 40-59中性 / 20-39偏弱 / <20弱势

## 交易纪律摘要

| 规则 | 条件 | 操作 |
|------|------|------|
| 入场 | ≥3/4信号满足 | 首次半仓（2.5万元） |
| 加仓 | 已有半仓+突破回踩确认 | 加至满仓（5万元） |
| 止损 | 单支亏损≥1000元 | 无条件止损 |
| 减仓 | SMA20下行 / 连3日失守SMA5 | 减仓或清仓 |
| 二选一 | 两支同时满足 | 选近10日跌幅更小、率先企稳的 |

## 注意事项

- 分析结果仅供研究参考，不构成投资建议
- 建议使用 temperature=0.1~0.2 以获得稳定输出
- 大模型可能计算能力有限，所有技术指标请自行计算后填入数据
