# ETF多资产全球轮动策略 — 结构化量化提示词 v2.0

## 概述

基于 `etf-running/v1-base.py` 中的多资产全球轮动策略，提供一套结构化量化分析提示词。你自行计算6支ETF的风险调整动量、ATR、SMA20状态、跨境溢价后，填入数据模板，发送给大模型生成周度量化日报。

## 策略核心逻辑（v1-base.py）

| 要素 | 规则 |
|------|------|
| **ETF池** | 6支权益：中证500、红利、纳指(跨境)、日经(跨境)、科创50、创业板 + 1支国债防御 |
| **核心评分** | 风险调整动量 = 20日收益率 ÷ 20日价格波动率（类夏普比率） |
| **趋势过滤** | 价格 > SMA20 **且** SMA20 未下行（走平或上行） |
| **跨境过滤** | 纳指/日经ETF 溢价率 > 1.5% 则剔除 |
| **仓位分配** | 风险平价：权重 ∝ 1/ATR，选前2名，单支≤50% |
| **波动率截断** | 全资产平均ATR > 3.5% → 40%转国债ETF防御 |
| **止损** | SMA20破位清仓；3倍ATR移动止损（自最高价计算）清仓 |
| **调仓频率** | 每周一 |

## 文件说明

| 文件 | 用途 |
|------|------|
| `momentum_prompt_config_v2.json` | 完整配置：策略说明、数据Schema、详细提示词、评分指南、ATR计算参考 |
| `momentum_daily_prompt_template_v2.txt` | 独立提示词模板，替换 `{DATA_INPUT}` 和 `{REPORT_DATE}` 后发送 |
| `momentum_data_input_template_v2.json` | 数据输入模板，按结构填入6支ETF的计算数据 |

## 使用流程

### 第一步：自行计算

需要计算以下指标（参考 v1-base.py 中的计算逻辑）：

**每支ETF：**
- 价格快照：收盘价、涨跌幅、成交量
- 风险调整动量得分：`return_20d / volatility_20d`
- 趋势过滤状态：`close > SMA20` 且 `SMA20[-1] >= SMA20[-2]`
- ATR指标：14日ATR值、ATR%
- 辅助动量：5日/10日收益、SMA5方向、量能比值

**跨境ETF 额外：**
- 溢价率 = (场内现价 - 基金净值) / 基金净值

**全资产层面：**
- 平均ATR% → 判断是否触发波动率截断 (>3.5%)
- 持仓诊断：最高价、3倍ATR止损线

### 第二步：填入数据模板

将计算结果填入 `momentum_data_input_template_v2.json`。

### 第三步：组装并发送

将 `momentum_daily_prompt_template_v2.txt` 中的：
- `{REPORT_DATE}` → 当前日期
- `{IS_REBALANCE_DAY}` → "是（周一调仓日）"或"否"
- `{DATA_INPUT}` → 第二步的完整JSON

连同 `momentum_prompt_config_v2.json` 中的 `system_prompt` 一起发送给大模型。

### 第四步：获取日报

大模型返回标准化 HTML 量化日报，含7个模块：
1. 📊 市场环境总览
2. 📈 风险调整动量排名
3. 📉 趋势过滤与均线系统
4. 🌊 波动率与风险敞口
5. 🎯 持仓诊断与止损监控
6. 💰 调仓建议与仓位分配
7. ⚠️ 风险提示与下周观察点

## v2.0 vs v1.0 主要变化

| 维度 | v1.0（旧策略） | v2.0（v1-base.py） |
|------|---------------|-------------------|
| ETF数量 | 2支 | 6支权益 + 1支国债 |
| 评分方式 | 4信号布尔判断 | 风险调整动量（收益/波动率） |
| 止损 | 固定1000元 | 3倍ATR动态移动止损 + SMA20破位 |
| 波动管理 | 无 | ATR > 3.5% → 40%国债防御 |
| 仓位 | 固定半仓/满仓 | 风险平价（反比ATR） |
| 跨境品种 | 无 | 纳指/日经，溢价检测 |
| 调仓频率 | 每日 | 每周一 |

## 注意事项

- 分析结果仅供研究参考，不构成投资建议
- 建议使用 temperature=0.1 获得稳定输出
- 所有技术指标请自行计算后填入，不要依赖大模型计算
